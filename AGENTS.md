# AGENTS.md — contributing to env-attr-cleaner

> **For an AI coding agent working on this repo.** This file is about developing **env-attr-cleaner itself** (the build-time cleaner). It is **not** a testing playbook — that lives in the separate [`test-casebook`](https://github.com/techmefr/test-casebook) repo. Do **not** pull the testing methodology in here: the dependency is one-way (the methodology knows about this tool, this tool knows nothing about the methodology).

## What this repo is

A pnpm-workspace monorepo publishing the cleaner that strips `data-test-*` (and configurable) attributes from production builds while keeping them in dev/test:

- `packages/unplugin` → **`env-attr-cleaner`** — the universal [unplugin](https://github.com/unjs/unplugin) (Vite, Webpack, Rollup, esbuild).
- `packages/bun` → **`env-attr-cleaner-bun`** — the Bun build plugin.
- `examples/` — one wiring example per framework (Nuxt, Vue, React, Next.js, Svelte, Astro, Bun).
- `docs/frameworks/` — per-framework wiring docs.

## Commands

```bash
pnpm install
pnpm build       # unbuild, all packages
pnpm test        # vitest run, all packages
pnpm typecheck   # tsc --noEmit, all packages
```

## Conventions

- **Don't edit `dist/`** — it's generated by `pnpm build` (and gitignored). Change `src/`.
- **Conventional commits, enforced by commitlint** — subject lowercase, ≤72 chars; body lines ≤100. A hook runs on `commit-msg`.
- **Keep bundler peer deps optional** (`peerDependenciesMeta`) — a consumer installs only the bundler they use.
- **The config shape is shared across every adapter** — `environments` keyed by `NODE_ENV`, arrays list the patterns to strip; every other `data-*` is preserved. Keep the unplugin and Bun implementations in sync.
- **The strip must be build-time only** — no runtime cost in shipped output.

## Boundaries

- Adding test scenarios, coverage rules, or the persona/mock doctrine → **not here**, that's `test-casebook`.
- Framework *wiring* docs live here (`docs/frameworks/`); the *testing guide* lives in `test-casebook`.
